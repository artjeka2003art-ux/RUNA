# RUNA — Architecture
# RUNA — Полная архитектура

## Видение продукта

Персональная prediction-система. AI строит живую модель человека через разговор,
формирует уникальный граф личности и моделирует параллельные сценарии будущего
через multi-agent симуляцию.

Вдохновлена MiroFish (OASIS/GraphRAG/Zep) — но переосмыслена для персональной
симуляции одного человека, а не социальной симуляции тысяч агентов.

---

## Слой 1 — Conversation Engine

**Назначение:** онбординг и ежедневный чекин

- Живой AI-диалог (Claude API) — не анкета, а разговор
- Извлечение сущностей: события, эмоции, паттерны, ценности, страхи, цели
- Автоматическая генерация уникальных сфер жизни из разговора
- Ежедневный 5-минутный чекин — обновление данных для графа

---

## Слой 2 — Personal Knowledge Graph

**Назначение:** долгосрочная память и граф личности

- GraphRAG — граф сущностей и связей между сферами, событиями, паттернами
- Zep — долгосрочная эпизодическая память агентов
- Векторная БД (Pinecone) — семантический поиск по истории пользователя
- Timeline store — хронологическая история изменений графа
- Обновляется ежедневно после каждого чекина

### Узлы графа

| Тип | Описание |
|-----|----------|
| Person | Центральный узел — сам пользователь |
| Sphere | Уникальные сферы жизни из онбординга |
| Event | Конкретные события: провал стартапа, переезд, начало терапии |
| Pattern | Повторяющиеся паттерны: "при стрессе изолируюсь" |
| Value | Ценности: "независимость важнее стабильности" |
| Blocker | Блоки и страхи: "боюсь снова провалиться" |
| Goal | Цели: декларируемые и латентные (из паттернов) |
| CheckIn | Ежедневный чекин с временной шкалой |

### Рёбра графа

| Тип | Пример |
|-----|--------|
| AFFECTS | Blocker → AFFECTS → Sphere (вес: 0.8) |
| CAUSED_BY | Pattern → CAUSED_BY → Event |
| CONFLICTS_WITH | Goal → CONFLICTS_WITH → Value |
| SUPPORTS | Value → SUPPORTS → Goal |
| CHANGED_ON | Sphere → CHANGED_ON → CheckIn |
| PREDICTS | Pattern + Blocker → PREDICTS → Scenario |

## Слой 3 — Multi-Agent Prediction Engine

**Назначение:** ядро prediction-симуляции

Четыре агента работают параллельно и независимо. Каждый читает граф
и формирует свою часть отчёта. Агенты не общаются между собой.

### Analyst Agent
- Вход: Personal Knowledge Graph + история чекинов
- Выход: обновлённые веса рёбер, новые паттерны, summary модели личности

### Scenario Agent × 3
- Вход: граф + задание построить сценарий для пути X
- Выход: сценарий с временной шкалой 30/90/365 дней, риски, первый шаг

### Predictor Agent
- Вход: все три сценария + граф личности
- Выход: вероятность каждого сценария (0–1) на основе паттернов

### Companion Agent
- Вход: граф + последние 7 чекинов + текущий сценарий
- Выход: вопрос дня, рекомендация дня, триггер обновления

**Главный принцип Companion Agent:**
❌ "Как ты сегодня по шкале 1-10?"
✅ "Три недели назад ты говорил что боишься снова провалиться.
    Сегодня ты молчал два дня. Что происходит?"

---

## Слой 4 — Life Score Engine

**Назначение:** ежедневный расчёт метрик

- Расчёт балла каждой сферы на основе данных графа и чекина
- Агрегация в единый Life Score (0–100)
- Детектор трендов — рост/падение каждой сферы за 7/30/90 дней
- Push-триггеры — уведомление если сфера падает N дней подряд
- История баллов — основа дофаминового удержания

---

## Слой 5 — Report & Interface

**Назначение:** отчёты и UI

- Prediction-отчёт: 3 сценария с временной шкалой, рисками и первым шагом
- Граф-визуализация сфер и связей
- Ежедневный дашборд: Life Score, динамика, рекомендация дня
- React Native (mobile-first) + веб-версия

## Хранилища данных

| Хранилище | Технология | Назначение |
|-----------|------------|------------|
| Graph DB | Neo4j | Основной граф личности. Cypher запросы. |
| Vector DB | Pinecone | Эмбеддинги всех текстовых данных. Семантический поиск. |
| Memory | Zep Cloud | Эпизодическая долгосрочная память агентов. |
| Time-series | TimescaleDB | История баллов Life Score. Тренды и графики. |
| Main DB | Supabase (PostgreSQL) | Пользователи, сессии, чекины, метаданные. |
| Cache | Redis | Кэш текущего состояния графа. TTL 24 часа. |

## Поток данных

### Онбординг (один раз)
1. Пользователь начинает разговор с Conversation Engine
2. Claude извлекает сущности: события, паттерны, ценности, блоки, цели
3. Claude генерирует уникальные сферы жизни (4–7 штук)
4. Analyst Agent строит первичный Personal Knowledge Graph
5. Scenario Agents × 3 генерируют первые три сценария
6. Predictor Agent рассчитывает вероятности
7. Пользователь видит граф и первый prediction-отчёт

### Ежедневный цикл
1. Пользователь проходит 5-минутный чекин с Companion Agent
2. Новые данные добавляются в граф
3. Analyst Agent пересчитывает веса рёбер
4. Life Score Engine пересчитывает баллы всех сфер
5. Если значимый сдвиг → Scenario Agents обновляют сценарии
6. Пользователь видит обновлённый дашборд с дельтой Life Score

### Триггерное обновление
- Сфера падает 3+ дня подряд → перегенерация сценария
- Пользователь совершил действие из сценария → обновление вероятностей
- Прошло 30 дней → полный пересчёт всех сценариев

## Дорожная карта фаз

### Фаза 1 — Conversation Engine (сейчас)
Conversation Agent + базовый граф + Companion Agent + Life Score + веб UI

### Фаза 2 — Full Prediction Engine
Scenario Agents × 3 + Predictor Agent + вероятности + полный GraphRAG

### Фаза 3 — Mobile + Retention
React Native + push-уведомления + триггерное обновление + история графа

### Фаза 4 — Scale & B2B
Блокчейн ownership данных + B2B API (HR, коучи, клиники)

### Что НЕ строим в MVP
- Блокчейн и ownership данных
- Мобильное приложение (сначала веб)
- Социальные функции
- B2B интеграции
- TimescaleDB (пока храним в PostgreSQL)
- Полный GraphRAG (упрощённая модель)

## Отличие от MiroFish

| Параметр | MiroFish | Runa |
|----------|----------|------|
| Объект симуляции | Общество (тысячи агентов) | Один человек |
| Семенной материал | Внешние данные | Личная история |
| Агенты | Виртуальные люди, взаимодействуют | Роли аналитика, независимы |
| Память (Zep) | О виртуальном мире | О реальной жизни пользователя |
| GraphRAG | Граф мира и персонажей | Граф личности |
| Выход | Отчёт о социальном явлении | Prediction-отчёт + Life Score |
| Лицензия | AGPL-3.0 | Собственная реализация |
