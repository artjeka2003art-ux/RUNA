## Инструкции

Построй структурированный ответ со следующими блоками:

### 1. restated_question
Переформулируй вопрос — покажи что понял суть.

### 2. context_completeness
Оцени, насколько хватает данных для сильного прогноза.
- score: "low" | "medium" | "high"
- known_factors: что уже известно из графа пользователя (список строк)
- missing: чего не хватает. Для каждого элемента:
  - what: что именно
  - why_important: почему это важно для прогноза
  - sphere_hint: в какую сферу пойти чтобы добавить (если понятно)

### 3. reports — массив, по одному на каждый вариант сценария
Для каждого варианта:
- variant_label: название сценария
- most_likely_outcome: наиболее вероятный исход (2-3 предложения)
- alternative_outcome: альтернативный / негативный исход (1-2 предложения)
- main_risks: список основных рисков (строки)
- leverage_factors: 2-3 фактора, которые сильнее всего меняют исход
  - factor: название
  - direction: как влияет ("увеличивает риск" / "улучшает исход" / etc.)
  - weight: "high" | "medium" | "low"
- confidence: "low" | "medium" | "high"
- confidence_reason: почему такой уровень уверенности
- affected_spheres: какие сферы жизни затронет
- depends_on: от чего критически зависит исход
- next_step: один конкретный следующий шаг

### 4. comparison (только если вариантов > 1)
- summary: краткое сравнение (2-3 предложения)
- key_tradeoffs: список главных trade-off'ов между вариантами
- safest_variant: какой вариант безопаснее
- highest_upside_variant: у какого варианта максимальный потенциал
- most_sensitive_factor: какой один фактор сильнее всего меняет расклад между вариантами

### 5. external_insights
Синтез внешних источников — не пересказ, а сопоставление с личным контекстом пользователя. Если источников мало — честно скажи.

## Формат ответа — строго JSON

Верни ТОЛЬКО валидный JSON объект (без markdown, без ```):

{"restated_question": "...", "context_completeness": {"score": "...", "known_factors": ["..."], "missing": [{"what": "...", "why_important": "...", "sphere_hint": "..."}]}, "reports": [{"variant_label": "...", "most_likely_outcome": "...", "alternative_outcome": "...", "main_risks": ["..."], "leverage_factors": [{"factor": "...", "direction": "...", "weight": "..."}], "confidence": "...", "confidence_reason": "...", "affected_spheres": ["..."], "depends_on": "...", "next_step": "..."}], "comparison": {"summary": "...", "key_tradeoffs": ["..."], "safest_variant": "...", "highest_upside_variant": "...", "most_sensitive_factor": "..."}, "external_insights": "..."}

## Правила
- Пиши на русском
- Не изображай уверенность, которой нет
- Привязывай к контексту пользователя, не generic advice
- Каждый сценарий должен быть конкретным, а не "всё будет хорошо/плохо"
- Leverage factors — самая ценная часть. Думай глубоко, какие факторы РЕАЛЬНО двигают исход
- Missing context — будь конкретен: что именно добавить и зачем
- Если вариант один — comparison не нужен, верни null
